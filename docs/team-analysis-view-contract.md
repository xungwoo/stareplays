# 팀 분석 뷰 계약

`/team-analysis`는 하나의 라우트와 컴포넌트를 공유하지만, `전체 시즌`과 `개별 시즌`은 서로 다른 제품 화면으로 관리한다. 수정할 때는 두 화면을 같은 카드/테이블 조합의 단순 필터 버전으로 보지 않는다.

## 데이터 스코프 불변식

- `전체 시즌`은 공식 3x3 전체 경기만으로 계산한다.
- `개별 시즌`은 선택한 `season_label`과 정확히 일치하는 경기만으로 계산한다.
- 개별 시즌의 `MVP`, `최강 종족(조합)`, `최약 종족`, `팀별 평균 피지컬 오각형`, `종족 역량 오각형`, `리플레이 피지컬 오각형`, `선수 역량 매트릭스`, `경기 전적 Raw`는 모두 선택 시즌 경기만 소스로 사용한다.
- 명시적 API 응답이 비어 있거나 실패한 경우 fixture/mock/전체 시즌 데이터로 fallback하지 않는다. 빈 상태나 오류 상태가 맞고, 다른 데이터로 보정하면 분석 오차다.
- 백엔드가 `season_label`을 필터링하더라도 프론트 loader/model 경계에서 같은 값을 한 번 더 검증한다. 이 방어선은 운영 API 회귀나 캐시 오염 시 교차 시즌 누수를 막기 위한 필수 가드다.
- 단일 종족 승률은 경기 수가 아니라 선수별 종족 출전 슬롯 기준이다. 한 3x3 경기에는 최대 6회의 종족 출전이 있으므로 `9-18, 27회 출전`처럼 시즌 경기 수보다 클 수 있다.
- 역량 오각형은 실제 수치 기반 비교 지표만 사용한다. `Winner Team`이라서 100, `Loser Team`이라서 0처럼 결과 라벨을 점수로 치환하는 축은 금지한다.
- 팀 평균 피지컬 오각형은 두 팀 상대 min/max banding을 쓰지 않는다. 각 축에서 높은 팀을 80으로 두고 다른 팀은 `낮은 평균 / 높은 평균 * 80`으로 점수화한다. 이렇게 해야 팀 간 실제 평균 비율은 보존하면서 차이를 과장하지 않는다.

## 전체 시즌 뷰

목적: 여러 시즌 전체를 기준으로 장기 성향, 모델 점수, 조합 안정성을 비교한다.

유지해야 할 정보:

- `최강 조합`: 전체 데이터에서 승률이 가장 좋은 선수 조합
- `평점 모델 원점수`: Bradley-Terry, TrueSkill 원점수 추이
- `핵심 인사이트`: BEST/위험 조합, 랜덤 성향, duo 궁합 등 장기 패턴. 선수 강점/약점 카드 바로 위에 배치한다.
- `승부 감각 오각형`: 승률, BT, TrueSkill, 주종, 팀 적응력
- `평점 모델 원점수`, `종족 조합`: 선수 역량 오각형 바로 아래에 배치한다.
- `선수 역량 매트릭스`: 전적, 승률, APM, 분당 생산, 종족별 승률, BT, TrueSkill, 강점, 약점. 조합/듀오 패널보다 위에 배치한다.
- `조합별 성적`: 전체 시즌 기준 선수 조합별 성적. `듀오 궁합`과 같은 row에서 각각 1열씩 차지한다.
- `듀오 궁합`: 전체 시즌 기준 함께 출전한 2인 조합 성적. `조합별 성적`과 같은 row에서 각각 1열씩 차지한다.

피해야 할 정보:

- `현재 팀 전적`: 개별 시즌 맥락의 즉시성 카드이므로 전체 시즌에서는 노출하지 않는다.

## 개별 시즌 뷰

목적: 선택한 시즌 안에서 현재 전적, 선수별 시즌 기여, 경기 원천 데이터를 빠르게 확인한다.

유지해야 할 정보:

- `현재 팀 전적`: 선택 시즌 안의 주요 팀/조합 현재 성적
- `선수 역량 매트릭스`: 승률, APM, 분당 유닛생산, P/T/Z전적, 강점, 약점
- `경기 전적 Raw`: 선택 시즌의 경기별 승리팀/패배팀/종족/APM. 항상 최신 경기순으로 표시한다.
- `플레이어 누적 승률 추이`: 선택 시즌 경기를 시간순으로 누적해 선수별 승률 흐름을 표시한다. `종족별 성적` 위에 배치한다.
- `조합별 성적`: 선택 시즌이면 선수 조합보다 종족 조합별 성적을 우선 표시한다. 각 row는 조합, 승패/승률, 표본 설명, 경기 수를 한 줄로 표현한다.
- `종족별 성적`: 선택 시즌의 P/T/Z 단일 종족 출전 슬롯 기준 승패와 승률. 왼쪽 열에서 `플레이어 누적 승률 추이` 아래에 배치하고, 오른쪽 열에는 `조합별 성적`을 배치한다.
- `선수 역량 오각형`: `종족 역량 오각형`, `리플레이 피지컬 오각형`을 선수별 패널 안에서 2열로 배치한다.
- `팀별 평균 피지컬 오각형`: 선택 시즌의 모든 승리 진영과 패배 진영을 각각 `Winner Team`, `Loser Team`으로 묶어 팀원 평균 피지컬을 비교한다. 축은 APM, EAPM, 명령효율, 분당 유닛생산, 자원 소모량이다. 선수 역량 오각형 패널 안에 넣지 않고 별도 1열 패널로 두며, 같은 row의 선수 역량 오각형 패널과 높이를 맞춘다.
- `리플레이 피지컬 오각형`: 팀 평균과 별개로 선수 개인별 피지컬 비교를 유지한다. 팀 평균 피지컬과 중복으로 보지 않는다.

피해야 할 정보:

- `최강 조합`: 시즌 표본에서는 장기 최강 조합으로 오해될 수 있으므로 노출하지 않는다.
- `평점 모델 원점수`: 같은 시즌/고정 팀 구조에서는 선수별 변별력이 낮다.
- `핵심 인사이트`: 전체 시즌 장기 패턴 프리셋과 섞이면 공감도가 낮다.
- `승부 감각 오각형`: 개별 시즌에서는 같은 팀 구성 때문에 BT/TrueSkill/승률 해석이 흔들린다.
- `선수별 종족 전적` 별도 테이블: 시즌 매트릭스의 P/T/Z전적 컬럼과 중복된다.

## 수정 체크리스트

팀 분석 화면을 수정할 때는 아래를 함께 확인한다.

1. 전체 시즌 테스트와 개별 시즌 테스트를 모두 갱신한다.
2. 새 카드나 테이블이 어느 뷰 계약에 속하는지 이 문서에 반영한다.
3. 공통 summary 필드를 추가하더라도 UI 노출은 `isAllSeasons` 기준으로 분리한다.
4. 전체 시즌에 모델/장기 패턴 지표를 추가할 때 개별 시즌에도 자동 노출하지 않는다.
5. 개별 시즌에 현재성/원천 전적 지표를 추가할 때 전체 시즌에도 자동 노출하지 않는다.
